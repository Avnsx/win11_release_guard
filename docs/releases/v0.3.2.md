## Compatibility

`pyproject.toml` keeps runtime dependencies limited to `cryptography>=41`. Test/tooling extras include `packaging>=24`, `pytest>=8`, and `tomli>=2` only on Python versions before 3.11 so repository tools that parse TOML still work on the declared 3.10 floor.

CI coverage is the compatibility proof for package users:

- `ubuntu-latest` on Python 3.10, 3.11, 3.12, 3.13, and 3.14.
- `windows-latest` on Python 3.10, 3.11, 3.12, 3.13, and 3.14.
- Matrix failures are not marked as allowed failures.

Release, PyPI publishing, Pages publishing, dependency, and Wiki-sync workflows keep Python 3.12 as their deterministic automation toolchain. They build universal wheel/sdist artifacts and rely on CI for interpreter compatibility coverage.

## Source Diagnostics

Source Diagnostics remain source-health evidence, not device-compliance verdict authority. Microsoft Release Health and Atom/Update History can be temporarily out of step. Preview, OOB, non-broad-target, unknown-family, and missing-KB Atom drift stays `notice` until reliable required-baseline evidence exists. Non-preview broad-target drift with an extracted KB and matching build/release evidence can be `warning`; notice-only drift does not trigger `source_drift_unresolved_after_24h`.

GitHub Issue sync remains workflow-only with `github.token` / `GITHUB_TOKEN`. It tracks real warning/error `source_diagnostics.events` only. Browser JavaScript renders static issue metadata and never creates, updates, or reopens issues.

If the PyPI project is absent before first publish, configure a Pending Trusted Publisher first. Pending Publisher does not reserve the package name; if the name is already controlled by another owner, stop and report instead of publishing.

## Unchanged Verdict Boundaries

- Signed public policy JSON plus detached signature remains the compliance verdict authority.
- Local Windows labels, WUA, Panther/setup logs, DISM, Event Logs, and Source Diagnostics remain diagnostic evidence.
- 25H2 remains the existing-device broad-fleet target.
- 26H1 remains excluded for existing-device targeting.
- `/api/v1` policy, signature, and manifest aliases stay backward-compatible.
